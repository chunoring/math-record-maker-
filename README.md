# 수학 수행평가 세특 메이커

수행평가와 학생별 관찰 자료를 정리하고, AI에서 세부능력 및 특기사항을 작성할 수 있는 프롬프트를 만드는 오프라인 웹앱입니다.

## 실행 방법

`index.html`을 더블클릭하면 브라우저에서 바로 실행됩니다. 설치나 인터넷 연결은 필요하지 않습니다.

## 사용 흐름

1. **수행평가 등록**에서 평가명, 과목, 활동 내용을 입력합니다.
2. 과목에 맞게 표시된 교육과정 성취기준을 하나 이상 선택하여 등록합니다.
3. **세특 만들기**에서 등록한 수행평가를 선택하고 학생 관찰 근거와 선택 항목을 입력합니다.
4. `수행평가별 프롬프트`를 새 AI 대화방에 한 번 붙여넣습니다.
5. 같은 평가의 학생마다 `AI용 학생 자료`를 복사하여 같은 대화방에 붙여넣습니다.
6. AI를 사용할 수 없을 때는 오프라인 보조 초안을 만들 수 있으며, 오타와 문맥을 반드시 검토합니다.

한 학급을 함께 작성할 때는 **학급 일괄 작성**을 사용합니다. 엑셀 표를 붙여넣거나 화면에서 직접 수정할 수 있으며, 학생 자료는 브라우저에 자동 저장됩니다. 앱은 AI 대화가 길어질 때 생기는 내용 혼합과 문체 변화를 줄이기 위해 5명 단위의 독립 프롬프트를 만듭니다. 각 묶음은 새 AI 대화방에 붙여넣고, 첫 결과 중 마음에 드는 문장을 `문체 기준 잠금`에 등록하면 다음 묶음도 비슷한 어조와 구성으로 작성할 수 있습니다.

교사용 학생 구분은 AI 프롬프트에 포함되지 않고 `학생 01`, `학생 02`처럼 익명 번호로 바뀝니다. 그래도 AI에 붙여넣기 전에 이름·학번 등 개인정보가 없는지 반드시 확인합니다.

화면 오른쪽 아래의 **? 사용방법**을 누르면 기본 작성 절차와 무료 생성형 AI로 다듬는 절차를 언제든 확인할 수 있습니다.

수행평가 정보는 브라우저의 로컬 저장소에 저장되며 외부로 전송되지 않습니다.

## 문장 작성 원칙

- 학생 관찰 근거를 가장 먼저 반영합니다.
- 주어로 `학생`, `그는`, `그가`, `그의`를 사용하지 않습니다.
- 문장을 현재형 명사형 어미(`~함`, `~음`, `~임`)로 정리합니다.
- 깨달음이나 느낌처럼 직접 관찰할 수 없는 내면 표현을 교사가 확인 가능한 행동 표현으로 바꿉니다.
- 성취 수준에 따라 평가어의 강도를 조절합니다.
- 선택한 성취기준은 등록·관리용으로만 사용하며 생성 문장에는 코드, 원문, 자동 추출한 개념명을 노출하지 않습니다.
- 관찰 근거가 짧으면 입력에 등장한 활동 단서를 바탕으로 구체화하고, 길면 반복 표현을 정리하여 핵심 행동 중심으로 다듬습니다.
